Specification

Language

The key words “MUST”, “MUST NOT”, “REQUIRED”, “SHALL”, “SHALL NOT”, “SHOULD”, “SHOULD NOT”, “RECOMMENDED”, “MAY”, and “OPTIONAL” in this document are to be interpreted as described in RFC 2119.

File location and contents

  • maker.json MUST reside in the root directory of a website's repository.
  • maker.json MUST include a single object represented as JSON, with key-value pairs according to the list below.

Fields

Required

The following information MUST be included:

Key Value
name A string specifying the location's name.
status A string indicating the status of the location. The value MUST be one of:
  • "active" - The location or community is active.
  • "inactive" - The location or community is inactive.
  • "planned" - The location or community is in planning stages.
type An array indicating the type of location/community. The value MUST be one of:
  • "Makerspace" - Your makerspace shop with fabrication tools, computer stations, and community space.
  • "Fab Lab" - As above, but ONLY those officially affiliated as Fab Labs.
  • "TechShop" - As above, but ONLY those officially affiliated as TechShops.
  • "Hackerspace" - A community space more focused on
  • "Coworking Space" - Open floor plans for startups and small businesses, not necessarily hackers only.
  • "Innovation Lab" - A space within larger organizations focused on emerging innovations or researching and development practices.
  • "University Lab" - Sharing many of the principles above but supported by an educational institution and may or may not be only open to students.
  • "Library" - Libraries as we know them but now offerings access to and workshops about DIY and fabrication topics (i.e. 3D printers).

Recommended

The following information SHOULD be included where applicable:

Key Value
source The source of this information. This could be a website, person, email address, or media source.
description A brief summary of the location and/or community. A good description should not reiterate other parts of the schema such as available tools or location but offer details on the mission, vision, history, and values.
address A string specifying the physical address of the location. This information SHOULD only include the street address or relative structures/business if that's not available. It SHOULD NOT other administrative information (to include separately, see below).
city A string specifying the city of the location.
stateRegionProvince A string specifying the state, region, or province of the location.
country A string specifying the country of the location.
latitude The longitude of your location. Optional to support virtual communities and events with changing locations. The value MUST be a decimal number ONLY to a minimum of two spaces.
longitude The longitude of your location. Optional to support virtual communities and events with changing locations. The value MUST be a decimal number ONLY to a minimum of two spaces.
phone A main point of contact email address. The object MUST be a valid phone number and is a string to allow for multiple formats and extensions.
website A string of the location/community's main website URL. The object MUST be a complete URL including http or https.
email A main point of contact email address. The object MUST be a valid email address.
id The URL describing the schema of the maker.json file.
social An array of name and URLs related to the location's offical use of social media. Facebook, Github, Instagram, etc. The object keys MUST be in lowercase (e.g., "facebook" or "instagram") and values MUST be a complete URL including http or https.

Optional

The following information MAY be included where applicable:

Key Value
photo The URL of an image file (e.g. logo or screenshot) to be used in a location listing. Value MUST be a complete URL.
tools Information about the tools available to members and/or the public. Examples include but are not limited to 3D printers, laser cutters, vinyl cutters, and computer workstations.
mailingList Information about joining a mailing list for your space. Value SHOULD be either an email address or URL.
openToExchanges A boolean (yes/no) statement if the space is open to exchanges.
dateFounded The date the space (physical or virtual) was founded. Value must be only a date and is stored in an ISO 8601 format.
numberOfMembers The average number of members for the space. Value MUST be only a number.
membershipFee A boolean (yes/no) response about if the space charges a fee to its members.
openToExchanges Brief details about the membership rates, if applicable. Value could contain a URL for aditional information.
openToResidencies A boolean (yes/no) answer to if the space is open to residencies. For easier upkeep, this answer SHOULD reflect if residencies are offered at all, not if there are currently openings.
sizeOfRooms The average size of the rooms.
