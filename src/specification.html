Specification

Language

The key words “MUST”, “MUST NOT”, “REQUIRED”, “SHALL”, “SHALL NOT”, “SHOULD”, “SHOULD NOT”, “RECOMMENDED”, “MAY”, and “OPTIONAL” in this document are to be interpreted as described in RFC 2119.

File location and contents

  • maker.json MUST reside in the root directory of a website's repository.
  • maker.json MUST include a single object represented as JSON, with key-value pairs according to the list below.

Fields

Required

The following fields MUST be included:

Key Value
name A string specifying the location's name.
address A string specifying the physical address of the location. Likely cases would include street, city, county/province/region, and area code. Where specific information is not available, just province is acceptable.
source A string specifying what organization or person supplied this information.
status A string indicating the status of the location. The value MUST be one of:
  • "Open" - The location is open for business
  • "Closed" - The location is permanently closed
  • "Temporarily Closed" - The location is temporarily closed.
type A string indicating the type of location. The value MUST be one of:
  • "Innovation Lab" - The location is open for business
  • "Makerspace" - The location is permanently closed
  • "FabLab" - The location is temporarily closed.
  • "TechShop" - Belonging to the official TechShop brand.

Strongly recommended

In addition to the above fields, the following fields SHOULD be included in every maker.json, where applicable:

Key Value
contact an object providing the individual or group fielding questions about the location. The object structure MUST contain a name property and SHOULD contain an email and/or url property.
description A description of the location usable as a summary.
homepage the location's URL, if it is a website, or the URL of a site describing the location.
id the URL describing the schema of the maker.json file.
social An array of name and URLs related to the location's offical use of social media. Facebook, Github, Instagram, etc.
tags An array of tag strings to help people search for your hlocation, which SHOULD have at least one item.

Recommended

Key Value
projects An array of data objects, highlighting and describing some or all of the projects happening in the location. Each data object MUST include name and url properties, giving the dataset's name and download URL, respectively. Each data object SHOULD also include a metadata property, providing the URL where the dataset's metadata can be read.
thumbnail the URL of an image file (e.g. logo or screenshot) to be used in a location listing
Partners An array of links

The following are commonly used fields that your maker.json MAY contain:

  • geography - an array of geography strings, indicating the city, state, county, or other geographic entity the location is relevant to.
  • links - an array of link strings, giving any additional webpage URLs relevant to your location.
  • partners - an array of partner objects, describing the primary individuals or groups that contributing to the location. Each partner object MUST contain a name property and SHOULD contain an email and/or url property.

Other

You MAY include other fields not listed above.
